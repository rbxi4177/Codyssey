1. Few-shot Learning의 개념

Few-shot Learning(FSL)은 인공지능, 특히 머신러닝과 딥러닝 분야에서 매우 적은 수의 학습 데이터만으로 새로운 작업을 학습하는 기술을 의미한다. 일반적인 딥러닝 모델은 높은 성능을 얻기 위해 수천~수백만 개의 학습 데이터를 필요로 한다. 하지만 현실에서는 충분한 데이터를 확보하기 어렵거나 데이터 수집 비용이 매우 높은 경우가 많다. 이러한 문제를 해결하기 위해 등장한 것이 Few-shot Learning이다.

'Few-shot'에서 'shot'은 하나의 학습 예시(example)를 의미한다. 즉, 모델이 새로운 개념을 이해하기 위해 제공받는 예시의 개수를 뜻한다.

예를 들어 고양이와 개를 구분하는 모델을 학습한다고 가정해 보자.

일반 딥러닝: 고양이 사진 10,000장, 개 사진 10,000장 이상 필요
Few-shot Learning: 고양이 사진 5장, 개 사진 5장만으로도 새로운 분류 가능

이는 사람이 새로운 사물을 몇 번만 보고도 구별할 수 있는 능력을 모방한 학습 방식이라고 볼 수 있다.

2. Few-shot Learning의 등장 배경

기존의 딥러닝은 대규모 데이터와 막대한 연산 자원을 필요로 한다. 그러나 실제 환경에서는 다음과 같은 문제가 발생한다.

① 데이터 부족

많은 분야에서는 충분한 데이터를 얻기 어렵다.

예시

희귀 질환 의료영상
희귀 동물 사진
산업 설비의 고장 데이터
신제품 이미지
② 데이터 수집 비용

데이터를 수집하고 라벨링(Labeling)하는 데는 많은 시간과 비용이 든다.

예를 들어 의료 영상은 전문 의사가 직접 판독해야 하므로 매우 높은 비용이 발생한다.

③ 새로운 클래스의 등장

이미 학습된 모델이라도 새로운 종류의 객체가 등장하면 다시 대규모 학습을 해야 한다.

Few-shot Learning은 적은 예시만으로 새로운 클래스를 추가할 수 있다는 장점이 있다.

3. Few-shot Learning의 기본 원리

Few-shot Learning은 기존의 딥러닝처럼 단순히 데이터를 많이 외우는 것이 아니라 학습하는 방법 자체를 학습(Learning to Learn) 하는 것이 핵심이다.

과정은 다음과 같다.

1단계 : 사전학습(Pre-training)

먼저 모델을 매우 큰 데이터셋으로 학습시킨다.

예를 들어

동물
자동차
사람
꽃

등 수백 개의 클래스로 학습한다.

이 과정에서 모델은

형태
색상
질감
특징

등을 일반적으로 이해하게 된다.

2단계 : 새로운 작업

새로운 클래스가 등장한다.

예를 들어

펭귄
코알라

사진이 각각 5장만 존재한다고 가정한다.

모델은 이전에 학습한 특징을 활용하여 새로운 클래스를 구분한다.

3단계 : 추론

새로운 이미지가 들어오면

기존에 학습한 특징과 Few-shot 예시를 비교하여 가장 가까운 클래스로 분류한다.

4. Few-shot Learning의 종류
(1) One-shot Learning

클래스당 단 하나의 예시만 제공된다.

예

고양이 사진 1장

↓

새로운 사진이 고양이인지 판단

가장 어려운 방식이다.

(2) Few-shot Learning

보통 2~10개의 예시를 제공한다.

예

고양이 사진 5장

개 사진 5장

↓

새로운 사진 분류

가장 많이 사용되는 방식이다.

(3) Zero-shot Learning

학습 예시가 전혀 없다.

대신

설명
속성
텍스트

만 제공된다.

예

"얼룩말은 말과 비슷하지만 줄무늬가 있다."

↓

모델은 설명만 보고 얼룩말을 인식한다.

최근의 대규모 언어 모델(LLM)은 이러한 Zero-shot 능력을 상당 수준 갖추고 있다.

5. Few-shot Learning의 주요 학습 방법
① Metric-based Learning

가장 널리 사용되는 방법이다.

새로운 데이터를 기존 데이터와 얼마나 비슷한지 계산한다.

대표 알고리즘

Siamese Network
Matching Network
Prototypical Network
Relation Network

예를 들어 얼굴 인식 시스템에서는

"새 얼굴이 기존 얼굴과 얼마나 비슷한가"

를 계산하여 사람을 구분한다.

② Optimization-based Learning

적은 데이터에서도 빠르게 학습할 수 있도록 최적화 방법을 개선한다.

대표 알고리즘

MAML(Model-Agnostic Meta-Learning)
Reptile

핵심은

"조금만 학습해도 빠르게 적응하는 모델"

을 만드는 것이다.

③ Model-based Learning

모델 자체에 기억(memory) 구조를 추가한다.

대표 모델

Memory-Augmented Neural Network

새로운 예시를 빠르게 기억하여 분류할 수 있다.

6. Meta Learning과의 관계

Few-shot Learning을 설명할 때 Meta Learning을 빼놓을 수 없다.

Meta Learning은

"학습하는 방법을 배우는 것"

이다.

즉,

기존 딥러닝

↓

데이터를 학습

Meta Learning

↓

학습하는 방법을 학습

Few-shot Learning은 Meta Learning을 활용하는 대표적인 응용 분야이다.

7. LLM에서의 Few-shot Prompting

최근에는 Few-shot이라는 용어가 대규모 언어 모델에서도 자주 사용된다.

예를 들어 ChatGPT에게

문장을 긍정/부정으로 분류하시오.

예시1
좋아요 → 긍정

예시2
별로예요 → 부정

예시3
최고입니다 → 긍정

입력
재미없어요

모델은 예시를 참고하여

부정

이라고 답한다.

이처럼 모델을 다시 학습시키지 않고, 프롬프트 안에 몇 개의 예시를 포함하여 원하는 작업을 수행하도록 하는 방법을 Few-shot Prompting이라고 한다.

8. Few-shot Learning의 장점
① 적은 데이터로 학습 가능

가장 큰 장점이다.

데이터가 거의 없어도 높은 성능을 기대할 수 있다.

② 비용 절감

데이터 수집 및 라벨링 비용이 크게 감소한다.

③ 빠른 적응

새로운 작업이나 클래스가 등장해도 빠르게 대응할 수 있다.

④ 실제 환경에 적합

현실에서는 데이터가 부족한 경우가 많으므로 실용성이 높다.

⑤ 다양한 분야 활용

이미지, 자연어 처리, 음성 인식 등 여러 분야에서 활용된다.

9. Few-shot Learning의 단점
① 성능 한계

데이터가 매우 적기 때문에 일반적인 대규모 학습보다 정확도가 낮을 수 있다.

② 과적합(Overfitting)

적은 데이터만 학습하면 특정 예시에 지나치게 맞춰지는 현상이 발생할 수 있다.

③ 일반화 어려움

새로운 환경에서 항상 좋은 성능을 보장하지는 않는다.

④ 복잡한 알고리즘

Meta Learning 등 고급 알고리즘이 필요하여 구현이 어렵다.

12. 미래 전망

Few-shot Learning은 데이터 부족 문제를 해결하는 핵심 기술로 주목받고 있다. 특히 생성형 AI와 대규모 언어 모델의 발전으로 적은 예시만으로도 다양한 작업을 수행하는 능력이 크게 향상되었다. 앞으로는 의료, 제조, 금융, 교육, 로봇공학 등 다양한 분야에서 활용 범위가 더욱 확대될 것으로 예상된다. 또한 Few-shot Learning은 전이학습(Transfer Learning), 메타러닝(Meta Learning), 검색 증강 생성(RAG) 등 다른 AI 기술과 결합하여 더욱 높은 성능과 효율성을 제공할 것으로 기대된다.

13. 결론

Few-shot Learning은 적은 양의 데이터만으로도 새로운 작업을 학습할 수 있도록 하는 머신러닝 기법으로, 데이터 확보가 어려운 현실적인 문제를 해결하기 위해 개발되었다. 기존의 지도학습이 대규모 데이터에 의존하는 것과 달리, Few-shot Learning은 사전학습과 메타러닝을 활용하여 새로운 환경에 빠르게 적응할 수 있다는 장점을 가진다. 최근에는 대규모 언어 모델에서 Few-shot Prompting의 형태로 널리 활용되며, 다양한 예시를 프롬프트에 포함해 모델의 응답 품질을 향상시키는 데 중요한 역할을 하고 있다. 비록 성능 한계와 과적합 등의 과제가 남아 있지만, Few-shot Learning은 데이터 효율성을 높이고 AI의 활용 범위를 넓히는 핵심 기술로 평가받으며 앞으로도 지속적인 연구와 발전이 이루어질 것으로 전망된다.

14. Few-shot Prompting 예시

Few-shot Prompting은 사용자가 원하는 작업에 대해 몇 개의 예시를 프롬프트에 먼저 제시한 뒤 새로운 입력을 처리하도록 하는 방법이다. 모델을 다시 학습시키지 않고도 예시를 통해 작업의 규칙과 형식을 이해하게 만든다는 점에서 효과적이다. 다음은 대표적인 예시이다.

예시 1. 감성 분석

사용자 프롬프트

다음 문장의 감성을 분류하세요.

예시 1
문장: 오늘 정말 즐거운 하루였다.
답: 긍정

예시 2
문장: 너무 실망스럽고 화가 난다.
답: 부정

예시 3
문장: 그냥 평범한 하루였다.
답: 중립

입력
문장: 음식이 정말 맛있었다.
답:

모델의 출력

긍정

설명

모델은 앞에서 제시된 세 개의 예시를 통해 감성 분류 기준을 학습하고, 새로운 문장에 동일한 기준을 적용한다.

예시 2. 문장 번역

사용자 프롬프트

다음을 영어로 번역하세요.

예시 1
안녕하세요.
→ Hello.

예시 2
오늘 날씨가 좋습니다.
→ The weather is nice today.

예시 3
저는 학생입니다.
→ I am a student.

입력
저는 축구를 좋아합니다.

모델의 출력

I like soccer.

설명

번역 규칙을 별도로 학습시키지 않아도 예시를 참고하여 자연스러운 번역을 수행한다.

예시 3. 텍스트 요약

사용자 프롬프트

다음 글을 한 문장으로 요약하세요.

예시 1
입력:
오늘은 날씨가 매우 맑았고 많은 사람들이 공원을 방문하였다.

출력:
많은 사람들이 맑은 날씨에 공원을 찾았다.

예시 2
입력:
신제품 스마트폰이 출시되었으며 카메라 성능이 크게 향상되었다.

출력:
카메라 성능이 향상된 신제품 스마트폰이 출시되었다.

입력:
인공지능 기술은 의료, 교육, 금융 등 다양한 분야에서 활용되고 있다.

출력:

모델의 출력

인공지능은 다양한 분야에서 활용되고 있다.

설명

Few-shot Prompting을 활용하면 원하는 요약 방식과 문장 형식을 예시를 통해 모델에 전달할 수 있다.

15. '되묻기(Clarification)' 흐름

Few-shot Prompting에서는 사용자의 요청이 모호하거나 필요한 정보가 부족한 경우, 모델이 바로 답을 생성하기보다 추가 정보를 요청하는 되묻기(Clarification) 과정을 거치면 더욱 정확한 결과를 얻을 수 있다. 되묻기는 사용자의 의도를 명확히 파악하여 오류를 줄이고, 보다 적절한 응답을 생성하는 데 중요한 역할을 한다.

예시 1. 번역 요청

사용자

이 문장을 번역해 줘.

AI(되묻기)

어떤 언어로 번역해 드릴까요? (예: 영어, 일본어, 중국어)

사용자

영어로 번역해 줘.

AI

번역 결과를 제공합니다.
예시 2. 요약 요청

사용자

이 글을 요약해 줘.

AI(되묻기)

몇 문장으로 요약할까요? 또는 핵심 내용만 간단히 요약할까요?

사용자

두 문장으로 요약해 줘.

AI

두 문장으로 요약된 결과를 제공합니다.
예시 3. 코드 생성 요청

사용자

로그인 프로그램을 만들어 줘.

AI(되묻기)

어떤 프로그래밍 언어를 사용할까요? (예: Python, Java, C++)

사용자

Python으로 작성해 줘.

AI

Python 코드와 설명을 제공합니다.
되묻기의 장점
사용자 의도를 정확하게 파악하여 잘못된 응답을 줄일 수 있다.
부족한 정보를 보완함으로써 더욱 적절한 결과를 생성할 수 있다.
사용자가 원하는 출력 형식(언어, 길이, 스타일 등)을 반영할 수 있다.

Few-shot Prompting과 되묻기의 관계

Few-shot Prompting은 모델에게 예시를 통해 작업 방식을 알려주는 기법이며, 되묻기(Clarification)는 필요한 정보가 부족할 때 추가 질문을 통해 사용자의 의도를 명확히 하는 대화 전략이다. 두 방법을 함께 사용하면 모델은 예시를 바탕으로 작업 방식을 이해하고, 부족한 정보를 보완한 뒤 최종 답변을 생성할 수 있어 응답의 정확성과 일관성을 더욱 높일 수 있다.
